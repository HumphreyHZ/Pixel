## Design Context

### Users
这个项目主要用于面试与作品集展示，核心受众是招聘方、面试官与设计/前端评审者，而非真实线上用户。因此，界面不仅要“能用”，更要在有限时间里快速体现出交互设计能力、产品思考、界面节奏与实现细节。

产品所展示的任务是“把专注、奖励、陪伴和探索串成一个完整闭环”。用户进入后，应该能迅速理解主线流程：开始专注，获得奖励，领取步数能量，兑换资源，再带着宠物继续探索或互动。

### Brand Personality
品牌气质以“治愈、聪明、有陪伴感”为核心。

“治愈”意味着整体视觉需要柔和、轻松、低压力，让人愿意停留；“聪明”意味着流程设计清晰、反馈准确、每一步都有明确的下一步引导；“有陪伴感”意味着宠物不是装饰，而是一个会回应、会提示、会把用户往前带的互动角色。

语气上应偏温柔、机灵、鼓励式，而不是命令式、说教式或冷冰冰的效率工具口吻。整体体验可以有游戏化奖励，但不应显得幼稚或吵闹。

### Aesthetic Direction
视觉方向以 PixPet 为主要参考，同时结合当前项目已存在的移动端单屏结构与像素宠物素材，形成“像素养成 + 专注工具 + 轻度冒险反馈”的界面语言。

从现有代码和参考信息推断，适合坚持以下方向：

- 移动端优先，信息密度克制，重点内容在首屏即可读懂
- 配色采用低饱和、偏暖和偏雾感的色板，如 sky、peach、sage、amber 一类的柔和色块
- 卡片、圆角、轻阴影和模糊层可以保留，但应服务于“柔和包裹感”，而不是追求炫技式玻璃效果
- 像素宠物是视觉锚点，既承担情绪价值，也承担流程推进中的反馈角色
- 游戏化反馈要明确可见，例如晶石、能量、体力、成就、地图推进，但布局上仍要保持干净和可讲述性

应避免的方向（基于当前定位推断）：

- 过于冷硬的企业效率工具风格
- 过于花哨、糖果色泛滥的儿童化游戏界面
- 只重装饰、不重主流程表达的视觉堆砌
- 让用户必须阅读大量说明后才能明白奖励闭环的复杂布局

### Design Principles
1. 先让人一眼看懂主循环，再去欣赏细节。
界面必须优先讲清楚“专注 -> 奖励 -> 兑换 -> 探索/陪伴”的闭环，避免把重要反馈埋在次级页面里。

2. 每次互动都要把用户自然推向下一步。
宠物回复、按钮文案、奖励提示和状态变化都应承担引导职责，让用户始终知道“接下来最值得做什么”。

3. 可爱是手段，不是目的。
像素宠物和养成元素应该增强记忆点与情感连接，但不能盖过专注工具本身的清晰度和可演示性。

4. 展示型项目要有清晰的交互证据。
关键状态变化、奖励增长、宠物反馈、路线切换和完成感都应足够明确，让面试官能快速看到交互设计与前端实现能力。

5. 温柔但不松散，游戏化但不幼稚。
整体氛围应保持治愈、聪明、陪伴感，同时维持节奏、秩序和专业完成度，避免过度卖萌或视觉噪音。
